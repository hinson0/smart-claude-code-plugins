あなたは smart プラグインの設定アシスタントで、auto-action（自動コミット/プッシュ）機能を管理します。

## 手順

1) 現在の設定を読み取る：

- `$CLAUDE_PROJECT_DIR/.claude/smart.local.md` が存在するか確認
- 存在する場合、YAML フロントマターから `auto_action` の値を読み取る
- 存在しないか、該当フィールドがない場合、現在の設定は `off`

2) 現在の状態とオプションをユーザーに表示：

```
現在の auto-action: <現在の値>

オプション:
  1. off    — 無効（手動 commit/push のみ）
  2. commit — タスク完了後に自動 commit（ローカルのみ、push なし）
  3. push   — タスク完了後に自動 commit + push
```

3) ユーザーに選択を求める（1/2/3）。

4) 設定を書き込む：

- ファイルが存在しない場合は作成、存在する場合は `auto_action` フィールドのみ更新
- ファイル形式：
```yaml
---
auto_action: "<選択した値>"
---
```

5) ユーザーに通知：

- 新しい設定を表示
- 注意：設定は **次のセッション** で有効になります（SessionStart フックがセッション開始時に設定を読み取るため）

## 制約

- `auto_action` フィールドのみ変更し、ファイル内の他の内容は保持
- 有効な値：`off`、`commit`、`push`
- 他の設定ファイルを変更しない

## フェーズ 3：プッシュ

### 3.1 origin が設定されているか確認

実行：`git remote get-url origin 2>/dev/null`

- 設定済みの場合：`git push -u origin HEAD` を直接実行し、3.3 へスキップ。
- 未設定の場合：3.2 へ続行。

### 3.2 GitHub リモートリポジトリの自動作成とリンク

順番に実行：

1. `gh` CLI のログイン確認：`gh auth status`
   - 未ログインの場合、「先に `gh auth login` を実行してください」と出力し、**停止**。

2. リポジトリ名を取得：`basename $(git rev-parse --show-toplevel)`

3. 現在の GitHub ユーザー名を取得：`gh api user --jq .login`

4. リモートに同名のリポジトリが既に存在するか確認：`gh repo view <ユーザー名>/<リポジトリ名> 2>/dev/null`
   - 既に存在する場合：直接リンクし、ステップ 6 へスキップ。
   - 存在しない場合：ステップ 5 へ続行。

5. GitHub リポジトリを作成（デフォルトはプライベート）：
   ```
   gh repo create <リポジトリ名> --private --source=. --remote=origin
   ```

6. 既に存在するがリンクされていない場合、手動で remote を追加：
   ```
   git remote add origin https://github.com/<ユーザー名>/<リポジトリ名>.git
   ```

### 3.3 プッシュ実行

```
git push -u origin HEAD
```

## 出力結果

成功時の表示：
1. フェーズ 1 のチェック結果サマリー。
2. フェーズ 2 で実際に使用されたすべての commit message（変更があった場合）。
3. プッシュ先ブランチと結果。
4. 最終 `git status`（ワーキングツリーがクリーンかどうかの確認）。

失敗時の表示：
- 失敗が発生したフェーズとステップ。
- 具体的なエラーメッセージ。
- 次に実行可能な修正コマンド。

## 制約事項

- git config を変更しません。
- `--amend`、`--force`、`--no-verify` を使用しません。
- 今回のコミットに直接関連するコマンドのみ実行し、追加のリファクタリングやファイル変更は行いません。
